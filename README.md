# sproto-rust

A pure-Rust implementation of the [sproto](https://github.com/cloudwu/sproto) binary serialization protocol.

Sproto is a compact, schema-driven serialization format designed for simplicity and efficiency, similar to Protocol Buffers but with a smaller feature set.

## Features

- **Binary schema loader** - Load pre-compiled binary schemas from C/Lua toolchain
- **Builder API** - Programmatic schema construction in pure Rust
- **Direct API** - Tag-based `StructEncoder`/`StructDecoder` for field-by-field encoding/decoding
- **Pack/Unpack** - Zero-packing compression for wire efficiency
- **RPC** - Request/response dispatch with session tracking
- **Lua binding** - `sproto-lua` crate exposes the Rust implementation as a Lua C module

## Installation

Add to your `Cargo.toml`:

```toml
[dependencies]
sproto = "0.1"
```

## Quick Start

### Building Schema in Rust

Use the Builder API to construct schemas programmatically:

```rust
use sproto::types::{Sproto, Field, FieldType};

let mut schema = Sproto::new();
schema.add_type("Person", vec![
    Field::new("name",   0, FieldType::String),
    Field::new("age",    1, FieldType::Integer),
    Field::new("active", 2, FieldType::Boolean),
]);
```

### Loading Binary Schema

Load pre-compiled binary schemas generated by the C/Lua toolchain:

```rust
let schema_bytes = std::fs::read("schema.bin").unwrap();
let schema = sproto::binary_schema::load_binary(&schema_bytes).unwrap();
```

### Encoding and Decoding

Use `StructEncoder`/`StructDecoder` for tag-based encoding and decoding:

```rust
use sproto::codec::{StructEncoder, StructDecoder};

let st = schema.get_type("Person").unwrap();

// Encode
let mut buf = Vec::new();
let mut enc = StructEncoder::new(&schema, st, &mut buf);
enc.set_string(0, "Alice").unwrap();
enc.set_integer(1, 30).unwrap();
enc.set_boolean(2, true).unwrap();
enc.finish();

// Decode
let mut dec = StructDecoder::new(&schema, st, &buf).unwrap();
while let Some(field) = dec.next_field().unwrap() {
    match field.tag() {
        0 => println!("name = {}", field.as_string().unwrap()),
        1 => println!("age = {}", field.as_integer().unwrap()),
        2 => println!("active = {}", field.as_boolean().unwrap()),
        _ => {}
    }
}
```

### Pack/Unpack Compression

```rust
use sproto::pack;

// Compress encoded data for transmission
let packed = pack::pack(&encoded);

// Decompress after receiving
let unpacked = pack::unpack(&packed).unwrap();
```

### RPC

```rust
use sproto::rpc::{Host, DispatchResult};

let schema = sproto::binary_schema::load_binary(&schema_bytes).unwrap();

// Create RPC host
let mut host = Host::new(schema.clone());

// Dispatch incoming packet
match host.dispatch(&packed_data).unwrap() {
    DispatchResult::Request { name, body, responder, ud } => {
        // Handle request, send response
        if let Some(resp) = responder {
            let response = resp.respond(&response_body, None).unwrap();
        }
    }
    DispatchResult::Response { session, body, ud } => {
        // Handle response
    }
}
```

## Schema Syntax

Sproto schemas are text files compiled to binary format by the C/Lua toolchain:

```sproto
.Person {
    name 0 : string
    age 1 : integer
    active 2 : boolean
    score 3 : double
    photo 4 : binary
    fpn 5 : integer(2)      # Fixed-point decimal
    phone 6 : PhoneNumber    # Nested struct
    friends 7 : *Person     # Array of structs
    tags 8 : *string        # Array of strings
}

# Protocol definition (RPC)
login 1 {
    request LoginRequest
    response LoginResponse
}

ping 2 {
    response PingResponse    # No request body
}

logout 3 {
    response nil             # Confirm, no response data
}

notify 4 {
    # One-way, no response
}
```

## Type Mappings

| Sproto Type | Rust Type |
|-------------|-----------|
| `integer` | `i64` |
| `boolean` | `bool` |
| `string` | `String` / `&str` |
| `binary` | `Vec<u8>` / `&[u8]` |
| `double` | `f64` |
| `integer(N)` | `i64` (scaled by 10^N) |
| `*type` | `Vec<T>` |
| `.Type` | nested struct |

## Wire Protocol Compatibility

This implementation is wire-compatible with the [reference C/Lua implementation](https://github.com/cloudwu/sproto). Binary data encoded by either implementation can be decoded by the other.
